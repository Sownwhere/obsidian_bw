
（motion_data_converter.py）

## 【总览】

该脚本是一个用于 **处理动作损失数据并生成用于人形机器人运动仿真的 .npz 数据文件** 的工具脚本，适用于 humanoid robot 运动复现。

---

## 【功能模块】

### 1. 四元数几何函数

- `quaternion_inverse(q)`：计算四元数逆
    
- `quaternion_multiply(q1, q2)`：四元数乘法
    
- `compute_angular_velocity(q_prev, q_next, dt)`：计算相邻四元数形成的角速度
    

### 2. Pinocchio 模型构造

- 通过 URDF 文件读入机器人模型，并添加 **free-flyer** 基座
    
- 应用于自由移动机器人（humanoid）
    

### 3. 主转换流程

#### 3.1 读取 CSV 数据

- 前 7 个值：root 的位置 (xyz) + 姿态 (quaternion)
    
- 其余值：各关节角度 (joint angles)
    

#### 3.2 重量采 (Resample)

- 把 90 fps 采样率转换为 60 fps
    
- 使用 `interp1d` 和 `Slerp` 分别接触位置/角度 和 四元数时间接近
    

#### 3.3 处理 root 和关节信息

- 根节点 root_data：位置 + 姿态
    
- 关节 joint_data：所有 DOF 角度
    
- 关节名 dof_names
    
- `z_offset`：增加 root 高度偏移
    

#### 3.6 计算关节速度

- 中心差分法 + 高斯滤波
    
- 降低速度旋动的随机性
    

#### 3.7-3.9 正向运动学 FK

- 通过 Pinocchio 运动学系统：
    
    - 得到每一帧的身体 link 位姿
        
    - 位置 + 四元数姿态
        

#### 3.10 计算 link 速度

- 线速度：位置差分 + 高斯滤波
    
- 角速度：四元数差分 → 分解成角速
    

#### 3.11 保存为 .npz

- 打包保存为 .npz 文件，包括:
    

```text
fps
body_names
dof_names
dof_positions
dof_velocities
body_positions
body_rotations
body_linear_velocities
body_angular_velocities
```

## 【总结】

### 用途

将人类动作数据转换为人形机器人可以进行学习/仿真/控制的结构化数据文件 (.npz)。

### 特点

- 利用 Pinocchio 实现高效的运动学计算
    
- 接入动作损失数据，转换为可控制格式
    
- 支持 root 类型模型 (floating base)
    
- 应用于 imitation learning, RL pretraining, motion replay
    

### 可扩展性

- 可增加 link 轨迹可视化功能
    
- 可增加对数据的错误检查
    
- 聚合成 CLI 命令行工具
