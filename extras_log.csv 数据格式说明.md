

该文件按帧记录机器人在仿真过程中的状态与传感器信息，每一行对应一帧。列的顺序如下：  

---

### 1. `ankle_angle`  
- **类型**：二维向量序列  
- **含义**：左右踝关节的角度（单位：弧度）  
- **形状**：`[1000, 2]`  
- **说明**：记录该帧内 1000 个采样点的左右踝关节角度。  

---

### 2. `ankle_torques`  
- **类型**：二维向量序列  
- **含义**：左右踝关节的实际控制力矩（单位：N·m）  
- **形状**：`[1000, 2]`  
- **说明**：记录该帧内 1000 个采样点的左右踝关节力矩。  

---

### 3. `left_imu_lin_acc`  
- **类型**：三维向量序列  
- **含义**：左腿 IMU 测得的线加速度（单位：m/s²，IMU 本体坐标系下）  
- **形状**：`[1000, 3]`  
- **说明**：记录该帧内 1000 个采样点的左腿线加速度。  

---

### 4. `left_imu_ang_vel`  
- **类型**：三维向量序列  
- **含义**：左腿 IMU 测得的角速度（单位：rad/s，IMU 本体坐标系下）  
- **形状**：`[1000, 3]`  
- **说明**：记录该帧内 1000 个采样点的左腿角速度。  

---

### 5. `right_imu_lin_acc`  
- **类型**：三维向量序列  
- **含义**：右腿 IMU 测得的线加速度（单位：m/s²，IMU 本体坐标系下）  
- **形状**：`[1000, 3]`  
- **说明**：记录该帧内 1000 个采样点的右腿线加速度。  

---

### 6. `right_imu_ang_vel`  
- **类型**：三维向量序列  
- **含义**：右腿 IMU 测得的角速度（单位：rad/s，IMU 本体坐标系下）  
- **形状**：`[1000, 3]`  
- **说明**：记录该帧内 1000 个采样点的右腿角速度。  

---

### 7. `left_ankle_hight`  
- **类型**：标量序列  
- **含义**：左踝关节在世界坐标系下的高度（单位：m）  
- **形状**：`[1000]`  
- **说明**：记录该帧内 1000 个采样点的高度。  

---

### 8. `right_ankle_hight`  
- **类型**：标量序列  
- **含义**：右踝关节在世界坐标系下的高度（单位：m）  
- **形状**：`[1000]`  
- **说明**：记录该帧内 1000 个采样点的高度。  

---

## 注意事项  
- 每一行是一个时间帧，字段顺序固定为上述 8 个。  
- 由于使用了 `tolist()`，每个字段在 CSV 中会被存成 **字符串形式的列表**，例如：  


"[[0.12, -0.08], [0.13, -0.07], ... 共1000项]", ...

- 读取时需要先解析字符串再转为 `numpy` 数组，例如：

import pandas as pd
import ast
import numpy as np

df = pd.read_csv("extras_log.csv", header=None)

# 取第一帧的 ankle_angle
ankle_angle_str = df.iloc[0, 0]  # 字符串
ankle_angle = np.array(ast.literal_eval(ankle_angle_str))  # 转成 (1000, 2) numpy 数组
