# 髋部外骨骼辅助算法 (DOFC)
[[Bokman等-Delayed Output Feedback Control for Gait Assistance With a Robotic Hip Exoskeleton.pdf]]

## 1. 传感器输入

*   **输入信号**: 左右髋关节角度
    *   `qr(t)`: 右髋关节角度
    *   `ql(t)`: 左髋关节角度
*   **传感器**: 安装在外骨骼上的角度传感器
*   **低通滤波器 (LPF)**: 原始角度信号 `qraw` 经过平滑处理：

    ```
    q_cur = (1 - alpha) * q_prv + alpha * q_raw,  0 < alpha < 1
    ```

    *   平滑因子 `alpha ≈ 0.05`
    *   低通滤波器引入约 0.13 秒的延迟

## 2. 输出状态定义

*   控制输出状态 `y(t)`:

    ```
    y(t) = sin(qr(t)) - sin(ql(t))
    ```
*   反映双腿运动差异（腿部摆动投影）
*   归一化范围：[-1, 1]

## 3. 延时反馈

*   辅助力矩通过延时反馈生成：

    ```
    tau(t) = kappa * y(t - Delta_t)
    ```

    *   `Delta_t`: 时间延迟（~0.15–0.35 秒，调整辅助时机）
    *   `kappa`: 反馈增益（调整辅助力矩大小）
*   该方法直接跟随用户运动，无需步态相位估计

## 4. 辅助力矩分配（左/右腿）

*   **右步态**（右髋屈曲，左髋伸展）：

    ```
    tau_r_des(t) = -tau(t) * beta_R * beta_Rhip
    tau_l_des(t) = tau(t) * beta_R * delta_R * beta_Lhip
    ```
*   **左步态**（左髋屈曲，右髋伸展）：

    ```
    tau_l_des(t) = tau(t) * beta_L * beta_Lhip
    tau_r_des(t) = -tau(t) * beta_L * delta_L * beta_Rhip
    ```
*   `beta`, `delta` 增益用于调整每条腿的屈曲/伸展辅助程度

## 5. 控制更新频率

*   控制周期：10 毫秒 (100 Hz)
*   确保辅助及时平滑

## 6. 稳定性分析

*   使用简化摆模型分析 `Delta_t` 和 `kappa` 的影响
*   适当的时间延迟即使在步态不规则的情况下也能稳定系统

## ✅ 主要特点

*   **无需步态相位估计**：基于髋关节角度差的直接控制
*   **适应不规则步态和环境变化**：如楼梯、坡道、突然停止
*   **参数可调**：
    *   `Delta_t`: 调整辅助时机
    *   `kappa`: 调整辅助力矩大小
*   **实现简单**：仅需髋部角度传感器和低通滤波器

# OPENEXO
[[Willians等-OpenExo: An open-­source modular exoskeleton to augment human function.pdf]]
## 1. **传感器输入**

髋关节助力主要依赖：

1. **髋关节角度传感器**
    
    - 实时测量髋关节屈曲/伸展角度 (q_{hip})。
        
2. **步态辅助传感器（可选）**
    
    - FSR 压力传感器放在鞋跟和鞋尖下，用于检测 **Heel Strike / Toe Off**，辅助识别步态阶段。
        
3. **扭矩传感器（可选）**
    
    - 髋助力通常开环，不用反馈，但可测量输出以验证性能。
        

> 所有数据通过 PCB 采集，频率 500 Hz。

---

## 2. **数据处理**

- **低通滤波**：滤除高频噪声，使角度信号平滑。
    
- **步态阶段识别**：
    
    - 通过压力传感器或角度变化判断步态阶段（屈曲、伸展、支撑）。
        

---

## 3. **控制策略**

### **开环控制（主要用于髋关节）**

- 输出 **预设扭矩曲线**，不依赖实时扭矩反馈。
    
- 例：
    
    - 屈曲期（抬腿）：输出 5 Nm
        
    - 伸展期（推进）：输出 5.5 Nm
        
- 精度约 RMSE 7%，符合实验要求。
    
- 步态触发方式：
    
    - 根据脚跟/脚尖接触（FSR）或髋关节角度阈值触发助力。
        

### **闭环控制（不常用）**

- 如果接入扭矩传感器：
    
    - 用 PD 控制或关节力矩比例控制，使实际扭矩跟随目标曲线。
        
    - 验证助力效果和安全性。
        

---

## 4. **算法流程（髋关节专用伪代码）**

```cpp
// 读取髋关节角度
q_hip = readHipAngle();

// 步态阶段判断
if (heelStrikeDetected()) {
    phase = HIP_FLEXION_PHASE;   // 抬腿阶段
} else if (toeOffDetected()) {
    phase = HIP_EXTENSION_PHASE; // 推进阶段
} else {
    phase = OTHER_PHASE;
}

// 低通滤波
q_hip_filtered = lowPassFilter(q_hip);

// 开环扭矩输出
if (phase == HIP_FLEXION_PHASE) {
    torque_cmd = 5.0; // Nm
} else if (phase == HIP_EXTENSION_PHASE) {
    torque_cmd = 5.5; // Nm
} else {
    torque_cmd = 0;
}

// 发送命令给电机
sendTorqueToMotor(torque_cmd);
```

---

## 5. **特点**

- **开环主控**，高响应快（Rise time 3 ms）。
    
- **不依赖闭环反馈**，简单可靠。
    
- **可模块化调整**：通过 SD 卡修改扭矩大小、步态阶段触发阈值，无需改源码。
    
- **功耗限制**：电池和电机温升是主要限制因素。

# Integral admittance shaping助力算法 - 髋关节
[[NagarajanAguirreGoswami-Integral admittance shaping-Accepted 1.pdf]]
## 一、参数确定

### 1. 系统参数（离线辨识）

| 参数 | 符号 | 典型值 | 获取方法 |
|------|------|--------|----------|
| 腿部转动惯量 | $I_h$ | 3.38 kg·m² | 身高体重查表 |
| 关节阻尼 | $b_h$ | 3.5 N·m·s/rad | 文献经验值 |
| 关节刚度 | $k_h$ | 54.7 N·m/rad | $k_h = I_h \omega_{nh}^2$ |
| 外骨骼惯量 | $I_e$ | 0.01178 kg·m² | 系统辨识 |
| 外骨骼阻尼 | $b_e$ | 0.345 N·m·s/rad | 系统辨识 |
| 外骨骼刚度 | $k_e$ | 0.339 N·m/rad | 系统辨识 |
| 耦合阻尼 | $b_c$ | 9.47 N·m·s/rad | 假设二阶系统 |
| 耦合刚度 | $k_c$ | 1905 N·m/rad | 假设二阶系统 |

### 2. 控制增益优化

**目标函数：**
$$\min_{\{K_\theta, K_\omega, K_\alpha, \omega_{lo}\}} \|A(\omega_f) - A_d\|^2 + w \cdot R(\omega_f)$$

**约束条件：**
- 稳定性：$GM(L_{heu}) > 1$
- 无源性：$\angle X_{heu}(j\omega) \in [-180^\circ, 0^\circ]$
- 舒适度：$|\zeta_{heu} - \zeta_h|/\zeta_h < \epsilon$

**增益计算：**
$$K_\theta = k_e - k_{ed}, \quad K_\omega = b_e - b_{de}, \quad K_\alpha = I_e - I_{ed}$$

> **注意**：目标阻抗参数 $k_{ed}, b_{de}, I_{ed} < 0$，因此控制增益为**正值**（正反馈）

---

## 二、控制方法

### 1. 输入

- **传感器**：霍尔效应传感器 → 关节角度 $\theta_e$
- **状态估计**：卡尔曼滤波 → $\hat{\theta}_e, \hat{\dot{\theta}}_e, \hat{\ddot{\theta}}_e$
- **滤波**：二阶 Butterworth 低通滤波（截止频率 $\omega_{lo}$）→ $\hat{\ddot{\theta}}'_e$

### 2. 控制律

$$\tau_e(t) = K_\alpha \cdot \hat{\ddot{\theta}}'_e(t) + K_\omega \cdot \hat{\dot{\theta}}_e(t) + K_\theta \cdot \hat{\theta}_e(t)$$

### 3. 输出

| 项目 | 说明 |
|------|------|
| **输出量** | 电机扭矩 $\tau_e$（限幅 ±6 Nm） |
| **输出方向** | 与角速度 $\dot{\theta}_e$ **同相**（正速度→正扭矩） |
| **物理效果** | 表现为"负阻尼"，抵消腿部自然阻尼 |

### 4. 输出时机

```
不需要：步态相位检测 / 意图识别 / if-else判断

实际机制：连续实时反馈控制（控制周期 >100Hz）

每一时刻 t：
  1. 读取角度 θe(t)
  2. 估计速度/加速度
  3. 滤波 + 代入控制律
  4. 输出扭矩 τe(t)
  5. 循环
```

---

## 三、实现流程

### 离线阶段
```
1. 获取用户身高体重 → 估算 {Ih, bh, kh}
2. 辨识外骨骼参数 {Ie, be, ke} + 耦合参数 {bc, kc}
3. 设定助力比 Ad（如 0.05）
4. 运行优化 → 得到 {Kθ, Kω, Kα, ωlo}
```

### 实时控制（伪代码）
```python
while device_on:
    θe = read_hall_sensor()
    {θ̂e, θ̇̂e, θ̈̂e} = kalman_filter(θe)
    θ̈̂e' = lowpass_filter(θ̈̂e, ωlo)
    τe = Kα*θ̈̂e' + Kω*θ̇̂e + Kθ*θ̂e
    send_to_motor(τe)
```

---

## 四、关键点

- **助力原理**：增大积分导纳幅值 → 运动放大/扭矩减小
- **正反馈**：补偿人体阻抗需 $K_\alpha, K_\omega, K_\theta > 0$
- **稳定性保障**：加速度信号低通滤波 + 增益裕度约束
- **任务无关性**：无需步态检测，连续反馈控制
- **发力时机**：关节运动的**每一时刻**，扭矩方向由当前运动状态实时决定









# 自适应负阻尼控制方法整理
[[Ramella-Adaptive Negative Damping Control for User-Dependent.pdf]]
## 一、控制方法核心原理

本研究采用**虚拟自适应负阻尼控制**（Virtual Adaptive Negative Damping Control）策略。其核心思想是通过在人体-外骨骼系统中注入能量来提供辅助，同时保持用户对运动的主动控制权。

控制律表达式为：
**Γexo = β · R · q̇hip**

其中：
- Γexo 为外骨骼输出的辅助扭矩
- β 为可调节的阻尼系数（辅助强度）
- R = [1 -1; 0 1] 为双腿耦合矩阵（基于compass-gait模型）
- q̇hip 为用户髋关节角速度向量

该方法的本质是：利用负阻尼特性将能量注入系统，而由于阻尼与关节速度成正比，辅助效果天然依赖于用户的主动运动，从而实现"用户主导、外骨骼跟随"的同步辅助。

---

## 二、输入信号

1. **髋关节角位置**（q1, q2）：分别表示站立腿和摆动腿的髋关节角度，通过电机编码器实时测量
2. **髋关节角速度**（q̇1, q̇2）：由编码器微分获得，经4Hz低通滤波处理后用于控制计算
3. **躯干姿态信息**：来自外骨骼躯干部分的IMU传感器，用于辅助步态相位判断
4. **步态事件检测**：通过髋关节角速度的符号变化识别站立腿与摆动腿（无需额外传感器）

> 注：滤波截止频率4Hz的确定依据：通过功率谱分析，在保证保留主要运动频率成分的前提下，有效抑制信号峰值噪声，同时避免静态站立时产生抖动辅助。

---

## 三、输出信号

1. **髋关节辅助扭矩**（Γexo）：
   - 输出形式：双侧髋关节的屈曲/伸展扭矩
   - 辅助时机：覆盖整个步态周期，在屈曲和伸展阶段均提供平滑过渡的辅助
   - 扭矩幅值：由β系数与实时关节速度共同决定，具有用户依赖性

2. **功率输出特性**：
   - 正功率占比＞98%，表明辅助扭矩与用户运动方向高度同步
   - 负功率（能量损耗）控制在总功率的2%以内

---

## 四、时间参数与实时性

1. **控制采样频率**：500 Hz（嵌入式控制器BeagleBoneBlack运行频率）
2. **扭矩计算方式**：每个采样周期实时计算，无固定波形模板
3. **优化更新周期**：贝叶斯优化每2秒窗口更新一次β值
4. **步态相位切换**：在脚跟触地瞬间即时完成站立腿/摆动腿的角色重置

---

## 五、关键参数确定方法

### 1. 阻尼系数 β 的确定
- **取值范围**：[1.0, 2.5] Nms/rad，通过预实验调优确定边界
- **自适应机制**：采用贝叶斯优化（Optuna框架）在线调整
- **优化目标函数**：
  ```
  fobj = 髋关节活动范围(ROM) − 平均加加速度(jerk)
  ```
  即：在增大运动范围的需求与抑制动作抖动之间寻求平衡

### 2. 信号滤波参数
- **滤波类型**：低通滤波
- **截止频率**：4 Hz
- **确定依据**：功率谱分析 + 预实验验证，兼顾信号保真与噪声抑制

### 3. 步态相位检测参数
- **检测依据**：髋关节角速度符号
  - 负速度（伸展）+ 对侧正速度（屈曲）→ 判定为站立腿
  - 反之则为摆动腿
- **优势**：避免使用角度位置导致的步态相位映射歧义

### 4. 扭矩安全限幅
- β值设置上下界，防止辅助过强干扰用户自然步态
- 确保外骨骼注入能量（Eexo）始终小于人体自身能量贡献，维持系统稳定性

---

## 六、多地形自适应机制

当用户在平地、上楼梯、下楼梯等不同场景间切换时：

1. **无需地形识别算法**：优化器仅依赖外骨骼本体感知的运动学信号
2. **自适应逻辑**：
   - 上楼梯：髋关节ROM增大 → 优化器自动提升β → 增强辅助
   - 平地行走：中等ROM与jerk → β保持中等水平
   - 下楼梯：动作更谨慎、jerk可能增大 → 优化器适度降低β
3. **过渡平滑性**：2秒优化窗口确保辅助强度变化连续，避免突变干扰

---

## 七、方法特点总结

- **输入**：仅依赖本体传感器（编码器+IMU），无需外部感知或生物电信号
- **输出**：与用户运动同步的连续扭矩，正功率占比高，能量传递高效
- **参数整定**：关键参数β通过贝叶斯优化在线自适应，减少人工调参
- **实时性**：500Hz控制频率 + 2秒优化周期，兼顾响应速度与计算负荷
- **通用性**：控制律形式简洁，无需复杂人体建模，便于迁移到不同用户和场景

该控制策略的核心优势在于：**以用户运动为驱动源，通过负阻尼机制实现"顺势辅助"，在降低代谢消耗的同时保持自然步态**。
